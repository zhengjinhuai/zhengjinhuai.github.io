# 找职位

  1. 职业规划：3年独当一面、独立负责系统；5年深耕发展的领域，5年后成为该领域专家

# 渠道

  1. 内推渠道：内推到自己所在团队，最好是团队leader a) 面试前能深入了解业务团队技术栈、业务细节 b) 进度跟进 c) 成员背书

  2. 猎头渠道

中高端岗位

a) 猎头必须是投递公司的s级合作方，或长期合作

  1. 企业招聘官网
  2. 招聘APP (BOSS直聘、猎聘)

# 调整简历

面向对象建简历

# 复盘

追问 - 分析 - 改进

# 信息收集

注意！！！尽可能收集信息（业务、技术栈、紧急程度、面试题、面试官、等等）

  1. 收集岗位信息 HR：部门、小组、业务 猎头：面试成功率、过往面试题、面试官风格 朋友：团队氛围、leader风格、加班情况、面试官风格 脉脉：收集差评 看准网：

  2. 判断岗位核心诉求

诉求：需要用A成本，请一个懂B技术，有C经验，具备D能力的人 技术诉求、能力诉求、业务经验诉求、成本诉求

技术栈判断 什么端：PC / 移动端 / 原生？ 用户群体：人群特点、人群与业务是否符合 团队组成：自己的核心程度，承担多大工作量

  1. 调整自己为岗位匹配者

# 面试前准备

自我介绍、复习基础、针对性技术准备、针对性项目准备

# 自我介绍

你好，我 xx 年毕业于 xx 学校，xx 专业，有 xx 年工作经验，主要在 xx 公司工作，技术栈主要是B技术，开发项目主要是C经验，在过去团队中，主要承担的任务是D能力

# 复习基础

第一层：数据结构 + 计算机网络（图解HTTP、网络是怎么链接的） + 设计模式 第二层：Javascript + HTML + CSS 第三层：框架原理 + 工程化 + 前端运维 第四层：nodejs + mysql

# 针对性技术准备

1、过往面试题目 覆盖 延展

2、面试官风格 喜欢问概念：八股文、刷题、背面试题库 喜欢问世间：多复盘项目

3、技术栈 框架/工具：了解对应原理，回顾实践中的问i，优化过的经验

# 针对性项目准备

突出能力、经验匹配度

项目描述：通过 xxx，解决 xxx 问题 个人负责： 项目难点：前端无记录差别录制会导致前端卡顿 业界方案：减少记录数据，使用 indexDB 存储、数据合并、web-worker 如何解决：卡顿本质是计算线程阻塞了渲染线程， 解决效果：从卡顿到近乎无感知

业务建议 对业务的了解 对行业竞品的了解 对该业务的经验 业务可能遇到的问题 对该业务的建议
