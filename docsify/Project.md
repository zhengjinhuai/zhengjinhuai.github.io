### 1.2 一些小亮点

- 路由懒加载
  - 只有用到该组件才去加载，避免没用到就加载的不必要的性能消耗
  - 在路由配置中，组件属性对应的是 import 引入组件的方法
  - 主要有三种方法实现：
    - vue 的异步组件：(resolve) => { require([], resolve)}
    - es6 的 import 方法：() => import( 需要异步加载的组件)
    - webpack 的 require.ensure: r => require.ensure([], () => r(require('@/components/list')), chunkName);
      - 这种是官方推荐的写的 按模块划分懒加载
      - 这种情况下，多个路由指定相同的 chunkName，会合并打包成一个 js 文件

- 移动端点击延迟事件

  - 原因：浏览器会在 touched 和 click 事件之间，等待 300-350ms，判断用户是否会进行双击手势以缩放文字
  - 解决方案：
    - 禁用缩放：<meta name="viewport" content="user-scaleable=no">，缺点：网页无法进行缩放
    - fastclick：在检测到 touchend 事件的时候，会通过 dom 自定义事件立即发出模拟 click 事件，并把浏览器在 300ms 之后真正的 click 事件阻止掉，缺点：脚本相对较大

- 首页组件使用 keepalive 进行组件缓存

  - 作用：用来缓存组件,避免多次加载相应的组件,减少性能消耗，简单一点来说就是从页面 1 链接到其他页面后回退到页面 1 不用在重新执行页面 1 的代码，只会从缓存中加载之前已经缓存的页面 1，这样可以减少加载时间及性能消耗，提高用户体验性
  - 实现方法：

    - 在首页组件中配置一个 mate 属性是 true
    - 用 keep-alive 标签将需要缓存的组件显示区包裹起来，在组件上使用$route.meta.keepAlive 判断组件是否显示，下面还有一个没有用 keep-alive 包裹的组件显示

  - 特点：
    - 重新进入组件不再重新渲染组件
    - 不进入组件普通的生命周期
    - 在进入组件和从组件离开有相关的方法，可以进行监听

- 商家数据懒加载
  - 首页首次只获取 4 条数据，从第 0 条开始
  - 给首页盒子添加滚动条监听事件
  - 当滚动到底部判断是否还有更多数据，有就去跟后端获取 4 条数据，从当前页面展示总条数开始

  - 显示哪些图片，再将图片的真实地址赋值

## 2. 难点与解决方法

2.1 CSS 的布局与居中问题

- 写 vue 界面的时候，渲染完成前调用 this.nextTick 方法无效，等到渲染完成再调用
- 提交图片 bug，有个问题：每次选择同一个图片，不调用 change 事件

  - 解决方案：每次 change 之后，将 input 内容清空，才会重新触发 change 事件

    2.2 其他问题

- v-if 和 v-show 的区别：
  - v-if ：可以根据表达式的值在 DOM 中生成或移除一个元素。
  - v-show:可以根据表达式的值来显示或者隐藏 HTML 元素。当 v-show 赋值为 false 时，元素被隐藏，此时查看代码时，该元素上会多一个内联样式 style=“display:none”。
- 重绘 repainted：页面中元素样式发生改变但不影响它在文档流中的位置时，浏览器会将新的样式赋给元素并重新绘制
- 回流 reflow，又称重排。当 render tree 中部分或者全部元素的布局或者几何属性发生改变时，浏览器重新渲染部分或全部文档称为回流
